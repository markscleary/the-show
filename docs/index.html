The Show

A programme runtime for creative work.
Open source. v1.0.0 — April 2026.

The Show runs creative production. Festivals, broadcasts, live events, anything that has a programme and a running order. Scenes run in dependency order. Approval gates wait for humans. Urgent contact reaches the operator across four channels. The show must go on.

Get started GitHub

What it is

Every programme is a YAML file. Every YAML file is a list of scenes. Each scene declares its inputs, outputs, dependencies, and how it produces its result — by calling an LLM, by waiting for a human, by reading a file, by sending a message. The runtime walks the dependency graph, runs scenes when they're ready, and stores everything in a state DB so the show can be paused, resumed, inspected, replayed.

Scenes can pause for human approval. The default timeout is seven days. Operators get notified across whichever channels they've configured — Telegram, email, SMS, WhatsApp. They reply with a single word. The show resumes.

Scenes can fail. Programmes can crash. The runtime carries on. Crash seams between scenes mean a programme that loses one scene doesn't lose the rest. The show must go on is the runtime's core promise, not its slogan.

What it isn't

The Show is not for code agents. It's not an enterprise automation runtime. It's not a SaaS platform. It's not where you go for compliance dashboards or RBAC.

The Show is for shows. Theatrical metaphor isn't decoration — it's what makes the runtime legible to creative operators who don't think in graph theory. Director, Stage Manager, House Manager, scenes, programmes, urgent contact. If you've worked in a venue, you already know how this works.

Install and run

pip install the-show
the-show validate examples/hello.yaml
the-show run examples/hello.yaml

hello.yaml runs in five seconds and exits clean. From there, the operator guide walks through every concept, the quickstart shows you a real programme, and the examples folder has the programme that built v1.0.0 itself.

About

The Show is part of Short+Sweet Agential, the agent infrastructure division of Short+Sweet International. Short+Sweet has run short-form performing arts festivals across fifteen countries for twenty-five years. The runtime is built for the kind of creative work S+S has always made — programmes, running orders, urgent decisions, the show must go on.
